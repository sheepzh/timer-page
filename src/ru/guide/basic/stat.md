## Измерения статистики

Время и количество посещений будут группироваться и агрегироваться по уникальному домену и дате. Это означает, что дата и домен составляют наименьшую статистическую детализацию.

И этот инструмент позволяет объединять базовые записи в соответствии с вашими собственными потребностями.

## Суммирование данных связанных сайтов

Как указано выше, эти данные агрегируются по домену. Например, `www.github.com` и `gist.github.com` считаются как 2 записи. Если вы хотите увидеть агрегированные данные с двух сайтов, вам нужно будет объединить их вручную.

Большинство страниц отображения данных поддерживают запросы на объединение. И вы можете настроить правила объединения на странице бэкенда.

### Как выглядят правила?

Правило состоит из двух частей: исходной части и целевой части. Исходная часть объявляет, какие сайты соответствуют правилу, а целевая часть определяет, как эти сайты объединяются. Например, `gist.github.com >>> github.com` — это обычное правило, левая часть `gist.github.com` — это исходная часть, а правая часть `github.com` — это целевая часть.

### Как определить исходную часть?

Исходная часть может быть конкретным доменным именем или выражением Ant. Ниже приведены некоторые примеры.

| Исходная часть   | Примеры соответствующих сайтов                                    |
| ---------------- | ----------------------------------------------------------------- |
| www.google.com   | Только `www.google.com` может соответствовать этому правилу       |
| www.google.com.* | `www.google.com.hk` `www.google.com.au`                           |
| \*\*.mit.edu     | `www.mit.edu` `libraries.mit.edu` `web.mit.edu` `foo.bar.mit.edu` |

### Как определить целевую часть?

Целевая часть может быть конкретным доменным именем, положительным целым числом или оставлена пустой. Они будут представлены по одному в сочетании с исходной частью в таблице ниже.

| Исходная часть    | Целевая часть | Вид                               | Примечание                                                                            |
| ----------------- | ------------- | --------------------------------- | ------------------------------------------------------------------------------------- |
| www.google.com.*  | google.com    | `www.google.com.* >>> google.com` | Сайты, соответствующие `www.google.com.*`, будут объединены в конкретный `google.com` |
| www.google.com.hk |               | `www.google.com.hk >>> Not Merge` | `www.google.com.hk` не будет объединен из-за пустой целевой части                     |
| \*\*.github.com   | 2             | `**.github.com >>> Merged(2)`     | Сайты, соответствующие `**.github.com`, будут объединены по первым 2 уровням домена   |

::: tip
Приведенных выше примеров достаточно для удовлетворения ваших потребностей в большинстве случаев. Для получения более подробной информации см. [полную документацию](https://github.com/sheepzh/timer#merger-rule).
:::
